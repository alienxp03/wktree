# Repository Instructions

- Currently this is still in dev, so there's no need to consider for backwards compatibilities.
- When making code changes, run both `make lint` and `make test` before finishing. If either cannot be run, mention that explicitly in the final response.
- For tmux related features, you can use `tmux capture-pane` or any tmux features to test any of the features.
- When any feature has been added, updated, removed, always make sure to update `wktree init` templates with latest changes. And README.md too.
